Стандарт описывает 4 уровня изоляции:
**Первый**: read uncommitted не реализован
**Второй**: Read Commited используется по умолчанию, предотвращает потерю изменения и грязное чтение(запрос видит незафиксированные изменения других транзакций)
Установить изоляцию
```
set transaction isolation level read commit; 
```
Посмотреть уровень изоляции
```
show transaction isolation;
```

Значения по умолчанию не всегда дают желаемого результата поэтому, где можно нужно указывать состояние **STABLE** 

**Третий**: Repeatable read
На момент начала транзакции создается снимок бд и работает с ним. Если данные в бд изменены, транзакция не увидит эти изменения.
```
BEGIN TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

**Четвертый**: Serializable (Сериализуемый), определяется одним абзацем, говорящем, что при параллельном выполнении несколько сериализуемых транзакций должны гарантированно выдавать такой же результат, как если бы они запускались по очереди в некотором порядке.

Стандарт описывает следующие особые условия, недопустимые для различных уровней изоляции:

**«грязное» чтение** 
(в postgres недопустим, но может существовать в других)
Транзакция читает данные, записанные параллельной незавершённой транзакцией.

**неповторяемое чтение**
Транзакция повторно читает те же данные, что и раньше, и обнаруживает, что они были изменены другой транзакцией (которая завершилась после первого чтения). Читает одни и те же данные и получает разные значения.

**фантомное чтение** 
(невозможно в Postgresql)
Транзакция повторно выполняет запрос, возвращающий набор строк для некоторого условия, и обнаруживает, что набор строк, удовлетворяющих условию, изменился из-за транзакции, завершившейся за это время.

**аномалия сериализации**
Результат успешной фиксации группы транзакций оказывается несогласованным при всевозможных вариантах исполнения этих транзакций по очереди.
